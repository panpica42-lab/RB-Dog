# 机械狗部署使用手册

本文面向现场部署、客户交付和日常使用，包含机械狗启动操作、WebSocket 网页控制台使用、Docker 镜像部署和常见问题排查。

## 1. 部署方式选择

| 方式 | 启动命令 | 适合场景 |
| --- | --- | --- |
| 本地运行 | `make run_ws_gateway` | 控制板已配置好 SDK 环境 |
| 本地模拟 | `make run_ws_mock` | 无机械狗时开发网页和 WS 逻辑 |
| Docker 真实模式 | `make docker_run_ws` | 批量部署、客户现场快速恢复 |
| Docker 模拟模式 | `make docker_run_mock` | 无机械狗时用容器开发和演示 |

真实控制机器狗时，推荐保留原厂掌机遥控器作为现场安全兜底。

## 2. 启动前检查

1. 控制板已进入 Ubuntu 系统。
2. 机器狗电量充足，周围留出安全空间。
3. 控制板有线网口连接机器狗扩展网口。
4. 控制板 Wi-Fi 和操作电脑/手机在同一个网络。
5. 默认网络参数正确。

默认网络参数：

| 项目 | 地址 |
| --- | --- |
| 机器狗 UDP | `192.168.96.2:8080` |
| 控制板有线网口 | `192.168.96.3` |
| 网页控制台端口 | `9001` |

## 3. 开机顺序

1. 长按机器狗背部电源键 3 秒以上开机。
2. 等待机器狗和控制板启动完成。
3. 听到“机器狗启动已完成”后，通过 SSH 登录控制板，例如：

```bash
ssh orangepi@10.0.50.236
```

4. 登录控制板后，再启动 SDK、WS 网关或 Docker 容器。

进入 SDK 目录：

```bash
cd /home/orangepi/quadruped_dev_sdk
```

## 4. 本地启动 WS 网关

真实机器狗模式：

```bash
make run_ws_gateway
```

正常输出应包含：

```text
Gateway mode: REAL
Robot UDP target: 192.168.96.2:8080
Web console: http://0.0.0.0:9001
UDP control listen: 0.0.0.0:9001
Robot UDP service is ready.
```

看到 `Robot UDP service is ready.` 表示控制板已经连通机器狗 UDP 服务。

无机械狗模拟模式：

```bash
make run_ws_mock
```

模拟模式不会等待机器狗 UDP 服务，会直接启动网页和 WebSocket 服务，并返回模拟状态、模拟 IMU 和模拟命令响应。

## 5. 浏览器访问

在控制板上查看 Wi-Fi IP：

```bash
ip -4 addr show wlan0
```

当前这台控制板的 Wi-Fi IP 示例：

```text
10.0.50.236
```

浏览器打开：

```text
http://10.0.50.236:9001
```

WebSocket 地址由网页自动使用：

```text
ws://10.0.50.236:9001/ws
```

如果现场 Wi-Fi IP 发生变化，只替换浏览器地址里的 IP，端口仍然是 `9001`。

## 6. 网页控制说明

网页连接成功后，右上角状态会显示“已连接”。真实模式下目标机器狗显示为 `192.168.96.2:8080`；模拟模式下目标显示为 `mock://quadruped`。

界面区域：

| 区域 | 说明 |
| --- | --- |
| 状态 | 显示电量、运动模式、运动模型和硬件故障状态 |
| 基础控制 | 提供复位、站起、趴下、急停、停止、停障、视频和动作命令 |
| IMU | 显示 Roll、Pitch、Yaw 和当前 IMU 帧编号 |
| RealSense RGB | 显示控制板直连 RealSense 的 RGB 画面 |
| RealSense 点云 | 手动开启后显示 RGB-D 生成的 3D 点云，支持鼠标/触控拖动旋转、滚轮缩放 |
| 摇杆 | 控制前后、左右平移和左右转向 |
| 消息 | 显示连接、命令响应和错误日志 |

常用操作：

| 操作 | 作用 |
| --- | --- |
| 复位 | 机器狗回零/初始化姿态 |
| 站起 | 从趴地状态站起 |
| 趴下 | 进入趴地状态 |
| 急停 | 立即急停 |
| 停止运动 | 速度归零，停止移动 |
| 停障开关 | 打开或关闭超声波停障 |
| 视频开关 | 打开或关闭机器狗原生视频流命令 |
| 越障模式/高速模式 | 切换运动模型 |
| 动作下拉 | 执行站立、打招呼、撒尿、跳跃、比心、拜年、原地模式 |
| 摇杆 | 控制前后、左右移动 |
| 左转/右转 | 按住持续转向，松开停止转向 |
| 转向速度 | 设置左转/右转速度 |
| 限速 | 限制摇杆速度比例 |

建议操作顺序：

1. 确认网页显示“已连接”。
2. 点击“复位”，等待姿态稳定。
3. 点击“站起”。
4. 先把“限速”和“转向速度”调低，例如 `0.35`。
5. 小幅拖动摇杆测试前进、后退、左右平移。
6. 按住“左转”或“右转”测试转向，松开后确认停止转向。
7. 观察状态、IMU 和 RGB 是否持续刷新；需要查看点云时点击“开启点云采集”，看完后及时关闭。
8. 必要时点击“停止运动”。

RealSense 画面说明：

1. 如果控制板没有接 RealSense，或缺少 `pyrealsense2`、`opencv-python`、`numpy`，网页会显示相机不可用，但不影响机器狗基础控制。
2. RGB 默认以较低码率推送，主要用于现场观察，不建议作为高帧率视频方案。
3. 点云数据量和 CPU 占用较大，默认不会生成点云；只有点击网页“开启点云采集”或 App 切到点云查看页时才会生成，离开页面、切后台或手动关闭会暂停采集。
4. 网络较差或现场不需要点云时，可以启动网关时加 `--no-pointcloud`，只保留 RGB。
5. 如果完全不需要相机，可启动 `tools/ws_gateway.py` 时加 `--no-camera`。

## 7. 停止与退出

停止运动：

1. 在网页点击“停止运动”。
2. 如需结束控制，先让机器狗“趴下”。
3. 在终端按 `Ctrl+C` 停止本地网关或 Docker 容器。

网关退出时会发送关闭 IMU 数据流命令。模拟模式下不会向真实机器狗发送任何 UDP 命令。

## 8. Docker 镜像内容

Docker 镜像内包含：

| 内容 | 路径 |
| --- | --- |
| C++ SDK 头文件 | `/opt/quadruped_dev_sdk/include` |
| C++ 静态库 | `/opt/quadruped_dev_sdk/lib/libquadruped_sdk.a` |
| 示例程序 | `/opt/quadruped_dev_sdk/bin` |
| WS 网关 | `/opt/quadruped_dev_sdk/tools/ws_gateway.py` |
| 网页控制台 | `/opt/quadruped_dev_sdk/web` |
| 客户文档 | `/opt/quadruped_dev_sdk/docs` |

不会打入镜像的内容包括原厂 PDF、掌机 APP 通信协议 DOCX、STEP 模型、抓包记录、`.git` 等敏感或非交付文件。

## 9. 构建 Docker 镜像

在 SDK 根目录执行：

```bash
cd /home/orangepi/quadruped_dev_sdk
make docker_build
```

如果当前用户没有 Docker daemon 权限，可以使用：

```bash
make docker_build DOCKER="sudo docker"
```

也可以把当前用户加入 `docker` 用户组，后续就不需要每次加 `sudo`：

```bash
sudo usermod -aG docker $USER
```

执行后退出当前 SSH 或终端，重新登录，再验证：

```bash
docker ps
```

如果不再提示 `/var/run/docker.sock` 权限不足，就可以直接使用：

```bash
make docker_build
```

等价 Docker 命令：

```bash
docker build -t quadruped-sdk:1.0.0 -t quadruped-sdk:latest .
```

## 10. Docker 真实机器狗模式

真实模式需要容器能访问控制板有线网口和机器狗 `192.168.96.2:8080`。推荐使用 Docker host 网络：

```bash
make docker_run_ws
```

如果需要 sudo：

```bash
make docker_run_ws DOCKER="sudo docker"
```

等价命令：

```bash
docker run --rm -it --network host quadruped-sdk:latest ws
```

启动后浏览器访问：

```text
http://<控制板 Wi-Fi IP>:9001
```

如果机器狗 IP 或端口不同：

```bash
docker run --rm -it --network host \
  -e DOG_IP=192.168.96.2 \
  -e DOG_PORT=8080 \
  -e LISTEN_PORT=9001 \
  quadruped-sdk:latest ws
```

## 11. Docker 无机械狗模拟模式

模拟模式不连接真实机器狗，适合客户没有硬件时开发网页和 WS 逻辑。

```bash
make docker_run_mock
```

如果需要 sudo：

```bash
make docker_run_mock DOCKER="sudo docker"
```

等价命令：

```bash
docker run --rm -it --network host \
  -e LISTEN_PORT=9101 \
  quadruped-sdk:latest ws-mock
```

访问：

```text
http://<控制板 Wi-Fi IP>:9101
```

模拟模式下网页命令格式与真实模式一致，WebSocket 地址为：

```text
ws://<控制板 Wi-Fi IP>:9101/ws
```

切换到真实机器狗时，客户网页代码不需要修改，只需要改启动命令和访问端口。

## 12. Docker Compose

真实机器狗模式：

```bash
docker compose up --build quadruped-ws
```

模拟模式：

```bash
docker compose --profile mock up --build quadruped-ws-mock
```

`docker-compose.yml` 中同样使用 `network_mode: host`。

## 13. 容器常用入口命令

容器默认命令是 `ws`，也可以手动指定：

| 命令 | 作用 |
| --- | --- |
| `ws` | 启动真实机器狗 WS 网关 |
| `ws-mock` | 启动模拟 WS 网关 |
| `example` | 运行自动动作示例 |
| `keyboard` | 运行键盘控制示例 |
| `imu` | 运行 IMU 示例 |
| `bash` | 进入容器 shell |

示例：

```bash
docker run --rm -it --network host quadruped-sdk:latest bash
docker run --rm -it --network host quadruped-sdk:latest keyboard
```

## 14. 部署到其他控制板

在新控制板上部署时，确认以下事项：

1. 控制板架构与镜像架构一致。香橙派一般为 ARM64，应在 ARM64 设备上构建，或使用 Docker Buildx 构建 ARM64 镜像。
2. 控制板有线网口配置为 `192.168.96.3/24`。
3. 机器狗地址为 `192.168.96.2:8080`。
4. 控制板 Wi-Fi 能被客户电脑或手机访问。
5. 运行容器时使用 `--network host`。

查看控制板 IP：

```bash
ip -4 addr
```

查看容器是否启动：

```bash
docker ps
```

如果执行 Docker 命令提示 `/var/run/docker.sock` 权限不足，可将当前用户加入 `docker` 用户组后重新登录，或临时使用 `sudo docker`。

临时使用 `sudo docker` 构建和运行：

```bash
make docker_build DOCKER="sudo docker"
make docker_run_ws DOCKER="sudo docker"
make docker_run_mock DOCKER="sudo docker"
```

查看网关日志：

```bash
docker logs <container_id>
```

## 15. 镜像导出和导入

构建完成后可导出镜像：

```bash
docker save quadruped-sdk:latest -o quadruped-sdk-latest.tar
```

在另一台控制板导入：

```bash
docker load -i quadruped-sdk-latest.tar
```

导入后运行：

```bash
docker run --rm -it --network host quadruped-sdk:latest ws
```

## 16. 常见问题

### 网关一直显示 `Waiting for robot UDP service...`

说明控制板没有收到机器狗 UDP 的 `PONG`。

检查：

1. 机器狗是否已完成开机语音提示。
2. 控制板和机器狗扩展网口的网线是否插好。
3. 控制板有线网口是否是 `192.168.96.3/24`。
4. 机器狗目标 IP 是否仍为 `192.168.96.2`。

### 浏览器打不开网页

检查：

1. 电脑/手机是否和控制板 Wi-Fi 在同一网络。
2. 控制板 Wi-Fi IP 是否变化，重新运行 `ip -4 addr show wlan0` 确认。
3. 网关进程或 Docker 容器是否还在运行。
4. 端口是否监听：

```bash
ss -ltnp | grep 9001
```

### Docker 提示 `/var/run/docker.sock` 权限不足

临时处理：

```bash
make docker_build DOCKER="sudo docker"
```

长期处理：

```bash
sudo usermod -aG docker $USER
```

然后退出 SSH，重新登录，再执行：

```bash
docker ps
```

### 网页已连接但机器狗不动

检查：

1. 机器狗是否处于站立状态。
2. 是否触发急停或故障。
3. 网页“限速”或“转向速度”是否太低。
4. 先点击“停止运动”，再点击“复位”和“站起”重试。

## 17. 安全注意事项

1. 首次测试务必低速、小幅移动。
2. 操作前让机器狗周围保持空旷。
3. 始终保留原厂掌机遥控器作为现场安全兜底。
4. 出现异常动作时，立即点击“急停”或使用原厂遥控器停止。
5. 同一时间不要启动多个真实控制网关同时控制同一台机器狗。
6. 模拟模式只能验证网页流程，不能验证真实运动效果。
