# 机械狗二次开发使用手册

本文面向需要基于控制板进行二次开发的客户，说明如何使用本项目提供的 C++ SDK 和 WebSocket 网页控制网关。

## 1. 开发方式选择

本项目提供两种二次开发方式：

| 开发方式 | 适合场景 | 主要技术 |
| --- | --- | --- |
| C++ SDK | 本地程序、算法控制、传感器数据处理、长期运行服务 | C++、UDP、jsoncpp |
| WebSocket 网页控制 | 网页互动、教学项目、小游戏、低代码演示界面 | HTML、CSS、JavaScript、WebSocket |

C++ SDK 直接运行在控制板上，通过 UDP 与机器狗通信。WebSocket 网页控制网关运行在控制板上，对浏览器提供 HTTP 和 WebSocket 服务，浏览器只需要发送 JSON 命令即可控制机器狗。

## 2. 硬件与网络连接

推荐将控制网络和调试网络分开：

1. 控制板有线网口连接机器狗扩展网口，用于机器狗控制。
2. 控制板 Wi-Fi 连接现场网络，用于 SSH、网页访问和客户调试。
3. 电脑或手机连接到与控制板相同的 Wi-Fi。
4. 原厂掌机遥控器保留作为现场安全兜底。

默认控制网络：

| 设备 | 地址 |
| --- | --- |
| 机器狗 | `192.168.96.2` |
| 控制板有线网口 | `192.168.96.3` |
| 机器狗 UDP 端口 | `8080` |

查看控制板 Wi-Fi IP：

```bash
ip -4 addr show wlan0
```

浏览器和 WebSocket 访问都使用控制板 Wi-Fi IP。例如当前控制板 Wi-Fi IP 为 `10.0.50.236` 时：

```text
网页地址: http://10.0.50.236:9001
WS 地址: ws://10.0.50.236:9001/ws
```

## 3. 开机与启动顺序

1. 用网线连接控制板和机器狗扩展网口。
2. 长按机器狗电源键 3 秒以上开机。
3. 等待机器狗和控制板启动完成。
4. 听到“机器狗启动已完成”后，电脑通过 SSH 登录控制板。
5. 进入项目目录后，再运行 SDK 程序或 WS 网关。

进入项目目录：

```bash
cd /home/orangepi/quadruped_dev_sdk
```

```bash

```


### 4.2 运行示例

自动动作示例：

```bash
./bin/example
```

键盘控制示例：

```bash
./bin/keyboard_test
```

IMU 数据示例：

```bash
./bin/imu_client_example
```

运行运动类示例前，请确保机器狗周围安全，并保留急停手段。

### 4.3 最小 C++ 控制流程

```cpp
#include "quadruped_sdk/udp_client.h"
#include "quadruped_sdk/json_parser.h"

int main() {
    quadruped_sdk::UDPClient client("192.168.96.2", 8080);

    if (!client.initialize()) {
        return 1;
    }

    if (!client.testConnection()) {
        return 1;
    }

    quadruped_sdk::JsonParser parser(std::move(client));
    parser.run();

    parser.handleResetCommand();
    sleep(3);

    parser.handleStandUpCommand();
    sleep(3);

    parser.sendDirectionVel(0.3, 0.0, 0.0);
    sleep(1);

    parser.sendDirectionVel(0.0, 0.0, 0.0);
    parser.handleLieDownCommand();

    parser.stop();
    return 0;
}
```

### 4.4 常用 C++ API

| API | 作用 |
| --- | --- |
| `client.initialize()` | 初始化 UDP 客户端 |
| `client.testConnection()` | 等待机器狗 UDP 服务就绪 |
| `parser.run()` | 启动后台接收和发送线程 |
| `parser.stop()` | 停止后台线程 |
| `parser.getDogStatus()` | 获取机器狗状态 |
| `parser.handleResetCommand()` | 复位回零 |
| `parser.handleStandUpCommand()` | 站起 |
| `parser.handleLieDownCommand()` | 趴下 |
| `parser.handleEmergencyCommand()` | 急停 |
| `parser.handleSwitchMnnCommand(mode)` | 切换运动模型 |
| `parser.handleLidarObstacleCommand(enable)` | 开关停障 |
| `parser.sendDirectionVel(frontback, leftright, turn)` | 三轴速度控制 |
| `parser.enableIMUStream()` | 开启 IMU 数据流 |
| `parser.disableIMUStream()` | 关闭 IMU 数据流 |
| `parser.getLatestIMUData()` | 获取最新 IMU 数据 |

运动控制方向：

| 参数 | 正数 | 负数 |
| --- | --- | --- |
| `frontback` | 前进 | 后退 |
| `leftright` | 左移 | 右移 |
| `turn` | 左转 | 右转 |

速度范围均为 `-1.0` 到 `1.0`。建议客户项目先使用 `0.1` 到 `0.4` 的低速范围进行调试。

### 4.5 状态字段

```cpp
quadruped_sdk::DogStatus status = parser.getDogStatus();
```

常用字段：

| 字段 | 含义 |
| --- | --- |
| `battery_level` | 电量 |
| `motion_mode` | 当前运动模式 |
| `velocity_level` | 速度档位 |
| `model` | 运动模型，`0` 越障模式，`1` 高速模式 |
| `hardware_error` | 硬件故障，`0` 正常，`1` 有故障 |
| `robot_number` | 机器狗编号 |
| `robot_type` | 机器狗型号 |

`motion_mode`：

| 值 | 含义 |
| --- | --- |
| `0` | 趴地未回零 |
| `1` | 趴地 |
| `2` | 站立 |
| `3` | 低身位 |
| `4` | 翻倒 |


访问：

```text

```

### 5.2 真实模式和模拟模式

| 模式 | 启动命令 | 是否需要机械狗 | 用途 |
| --- | --- | --- | --- |
| 真实模式 | `make run_ws_gateway` | 需要 | 控制真实机器狗 |
| 模拟模式 | `make run_ws_mock` | 不需要 | 客户无硬件开发网页和 WS 逻辑 |

模拟模式下：

1. 网页访问方式不变。
2. WebSocket 地址仍然是 `/ws`。
3. 控制命令格式不变。
4. 网关会返回模拟状态、模拟 IMU 和模拟命令响应。
5. 切换到真实机器狗时，客户网页代码不需要修改，只需要改启动命令。

模拟模式不能验证真实运动效果、真实传感器数据和真实动作编号表现。

### 5.3 相机和点云参数

WS 网关默认会尝试启动控制板直连的 RealSense RGB/depth 流。RGB 会持续低码率推送；点云计算默认关闭，只有网页手动点击“开启点云采集”、App 切到点云查看页，或 WebSocket 客户端发送点云开启命令后才会生成和广播点云。

常用参数：

| 参数 | 默认值 | 说明 |
| --- | --- | --- |
| `--no-camera` | 关闭 | 禁用 RealSense RGB/depth 推流 |
| `--no-pointcloud` | 关闭 | 保留 RGB，彻底禁用点云能力 |
| `--camera-width` | `640` | RGB/depth 宽度 |
| `--camera-height` | `480` | RGB/depth 高度 |
| `--camera-fps` | `15` | RealSense 采集帧率 |
| `--camera-publish-fps` | `5` | WebSocket RGB 发布帧率 |
| `--camera-jpeg-quality` | `45` | RGB JPEG 质量 |
| `--pointcloud-fps` | `1` | 点云开启后的发布帧率 |
| `--pointcloud-max-points` | `20000` | 单帧点云最大点数 |

示例：

```bash
python3 tools/ws_gateway.py --no-pointcloud
python3 tools/ws_gateway.py --no-camera
python3 tools/ws_gateway.py --camera-publish-fps 3 --camera-jpeg-quality 35
```

WebSocket 手动开启/关闭点云：

```json
{"type":"pointcloud","enabled":true}
{"type":"pointcloud","enabled":false}
```

建议客户端只在用户正在查看点云时开启，切换页面、退后台、断开连接前都发送关闭命令；客户端断开后网关也会自动取消该连接的点云订阅。没有任何客户端订阅点云时，网关不会执行点云计算。

如果控制板没有接 RealSense，或缺少 `pyrealsense2`、`opencv-python`、`numpy`，网关会通过 `camera_status` 返回错误信息，基础控制命令仍可继续使用。

### 5.4 WS 网关的作用

WS 网关做了三件事：

1. 提供网页静态文件访问，默认目录是 `web/`。
2. 提供 WebSocket 接口 `/ws`，接收浏览器发来的 JSON 控制命令。
3. 将网页命令转换为机器狗 UDP 协议，转发到 `192.168.96.2:8080`。
4. 可选读取 RealSense RGB/depth，并通过 WebSocket 广播 RGB 图像和点云。

客户做网页二开时，一般只需要修改或新增 `web/` 目录下的网页文件，不需要改 UDP 协议。

### 5.5 网页项目放置方式

当前控制台页面：

```text
web/index.html
```

客户可以新增自己的页面，例如：

```text
web/projects/game.html
web/projects/class-demo.html
web/projects/block-programming.html
```

访问方式：

```text
http://<控制板 Wi-Fi IP>:9001/projects/game.html
```

注意：当前静态服务器不会自动把目录映射到目录下的 `index.html`。因此建议访问具体 HTML 文件路径。

### 5.6 网页连接 WebSocket

网页中使用当前页面所在主机连接 WS：

```html
<script>
const ws = new WebSocket(`ws://${location.host}/ws`);

function send(command) {
  if (ws.readyState === WebSocket.OPEN) {
    ws.send(JSON.stringify(command));
  }
}
</script>
```

如果页面使用 HTTPS 部署，应改成 `wss://`。本项目控制板本地服务默认使用 HTTP，因此默认是 `ws://`。

### 5.7 网页发送控制命令

基础命令：

```js
send({ type: "reset" });      // 复位回零
send({ type: "stand" });      // 站起
send({ type: "lie" });        // 趴下
send({ type: "stop" });       // 停止运动
send({ type: "emergency" });  // 急停
```

三轴运动：

```js
send({
  type: "move",
  frontback: 0.3,
  leftright: 0,
  turn: 0
});
```

方向说明：

| 字段 | 正数 | 负数 |
| --- | --- | --- |
| `frontback` | 前进 | 后退 |
| `leftright` | 左移 | 右移 |
| `turn` | 左转 | 右转 |

取值范围均为 `-1.0` 到 `1.0`。网页小游戏中建议先限制到 `-0.4` 到 `0.4`。

切换运动模型：

```js
send({ type: "model", value: 0 }); // 越障模式
send({ type: "model", value: 1 }); // 高速模式
```

停障开关：

```js
send({ type: "obstacle", enabled: true });
send({ type: "obstacle", enabled: false });
```

视频开关：

```js
send({ type: "video", enabled: true });
send({ type: "video", enabled: false });
```

动作命令：

```js
send({ type: "action", code: 11 });
```

当前网页控制台中使用的动作编号：

| code | 动作 |
| --- | --- |
| `11` | 打招呼 |
| `5` | 撒尿 |
| `2` | 跳跃 |
| `14` | 比心 |
| `15` | 拜年 |

当前 `web/index.html` 控制台提供的界面能力：

| 区域 | 能力 |
| --- | --- |
| 状态 | 电量、运动模式、运动模型、硬件故障 |
| 基础控制 | 复位、站起、趴下、急停、停止、停障、视频、运动模型、动作下拉 |
| IMU | Roll、Pitch、Yaw、Frame |
| RealSense RGB | 展示 `camera_frame` 中的 JPEG 图像 |
| RealSense 点云 | 手动开启后展示 `pointcloud_frame` 中的 `xyzrgb_flat` 点云，支持拖动旋转和滚轮缩放 |
| 摇杆 | `move.frontback`、`move.leftright`、`move.turn` 三轴速度控制 |
| 消息 | 连接、发送命令、命令响应和错误日志 |

### 5.8 网页接收状态、IMU、RGB 和点云

连接成功后，网关会发送 `hello` 消息：

```json
{
  "type": "hello",
  "dog": "192.168.96.2:8080",
  "mock": false,
  "camera": true,
  "pointcloud": true,
  "robot_ready": true,
  "status": {},
  "imu": {}
}
```

模拟模式下 `hello.mock` 为 `true`，`dog` 为 `mock://quadruped`。`hello.pointcloud` 表示网关是否具备点云能力，不表示当前正在采集点云；当前是否采集由客户端发送的 `pointcloud` 开关命令决定。

状态更新：

```json
{
  "type": "status",
  "status": {
    "battery": 51,
    "motion_mode": 2,
    "model": 0,
    "hardware_error": 0
  }
}
```

快照更新：

```json
{
  "type": "snapshot",
  "status": {},
  "imu": {},
  "clients": 1,
  "time": 1778190000.0
}
```

IMU 更新：

```json
{
  "type": "imu",
  "imu": {
    "frame_id": "imu_link",
    "orientation": {},
    "angular_velocity": {},
    "linear_acceleration": {}
  }
}
```

RGB 状态：

```json
{
  "type": "camera_status",
  "ok": true,
  "message": "RealSense RGB/depth stream started",
  "time": 1778190000.0
}
```

RGB 帧：

```json
{
  "type": "camera_frame",
  "format": "jpeg",
  "encoding": "base64",
  "width": 640,
  "height": 480,
  "time": 1778190000.0,
  "data": "/9j/..."
}
```

网页端可直接拼成 data URL：

```js
img.src = `data:image/jpeg;base64,${message.data}`;
```

点云状态：

```json
{
  "type": "pointcloud_status",
  "ok": false,
  "message": "Point cloud has no valid depth points",
  "time": 1778190000.0
}
```

点云默认不会刷新。需要查看时先发送 `{"type":"pointcloud","enabled":true}`，网关会返回 `pointcloud_status` 并开始广播 `pointcloud_frame`；当前网页控制台会根据收到的点数显示“3D 点云 N 点”。查看结束后发送 `{"type":"pointcloud","enabled":false}` 暂停点云计算。

点云帧：

```json
{
  "type": "pointcloud_frame",
  "format": "xyzrgb_flat",
  "width": 640,
  "height": 480,
  "count": 20000,
  "time": 1778190000.0,
  "points": [0.12, -0.08, 1.35, 255, 180, 90]
}
```

`points` 是扁平数组，每 6 个值表示一个点：

```text
x, y, z, r, g, b
```

其中 `x/y/z` 单位为米，`r/g/b` 范围为 `0` 到 `255`。

网页可以按消息类型处理：

```js
ws.onmessage = event => {
  const message = JSON.parse(event.data);

  if (message.type === "status") {
    console.log(message.status);
  }

  if (message.type === "imu") {
    console.log(message.imu);
  }

  if (message.type === "camera_frame") {
    image.src = `data:image/jpeg;base64,${message.data}`;
  }

  if (message.type === "pointcloud_frame") {
    renderPointcloud(message.points);
  }
};

// 只有用户打开点云视图时才开启；离开视图或页面隐藏时关闭。
ws.send(JSON.stringify({ type: "pointcloud", enabled: true }));
ws.send(JSON.stringify({ type: "pointcloud", enabled: false }));
```

### 5.9 网页小游戏的推荐控制流程

网页小游戏或互动项目建议遵循以下流程：

1. 连接 WebSocket。
2. 确认页面显示已连接。
3. 发送 `reset`。
4. 等待 2 到 4 秒。
5. 发送 `stand`。
6. 等待机器狗站稳。
7. 执行低速动作序列。
8. 每段动作结束后发送 `stop`。
9. 游戏结束发送 `lie`。

示例动作序列：

```js
function wait(ms) {
  return new Promise(resolve => setTimeout(resolve, ms));
}

async function runSequence() {
  send({ type: "reset" });
  await wait(3000);

  send({ type: "stand" });
  await wait(3000);

  send({ type: "move", frontback: 0.25, leftright: 0, turn: 0 });
  await wait(1000);

  send({ type: "stop" });
  await wait(500);

  send({ type: "move", frontback: 0, leftright: 0, turn: 0.25 });
  await wait(1000);

  send({ type: "stop" });
}
```

## 6. HTTP 与 UDP 控制入口

WS 网关默认同时监听：

```text
HTTP/WebSocket: 0.0.0.0:9001
UDP 控制入口: 0.0.0.0:9001
```

HTTP/WebSocket 用于浏览器控制。

UDP 控制入口可用于其他程序发送同样的 JSON 命令。例如发送：

```json
{"type":"stop"}
```

如果 UDP 收到的是带 `type` 字段的 JSON，网关会按网页命令处理；如果收到的是原始机器狗协议 JSON，网关会直接转发到机器狗。

## 8. 常见问题

### 8.1 `Waiting for robot UDP service...`

说明控制板没有收到机器狗 UDP 服务响应。

检查：

1. 机器狗是否已完成开机。
2. 控制板与机器狗扩展网口是否连接。
3. 控制板有线网口是否为 `192.168.96.3/24`。
4. 机器狗 IP 是否为 `192.168.96.2`。

### 8.2 浏览器打不开网页

检查：

1. 电脑/手机是否与控制板在同一 Wi-Fi。
2. 控制板 Wi-Fi IP 是否变化。
3. 网关是否正在运行。
4. 端口是否监听：

```bash
ss -ltnp | grep 9001
```

### 8.3 网页已连接但机器狗不动

检查：

1. 机器狗是否处于站立状态。
2. 是否触发急停或硬件故障。
3. 速度值是否太小。
4. 先发送 `stop`，再重新 `reset` 和 `stand`。

### 8.4 C++ 程序连接失败

检查：

1. `client.initialize()` 是否成功。
2. `client.testConnection()` 是否返回 `true`。
3. 机器狗 UDP 地址是否为 `192.168.96.2:8080`。
4. 有线控制网口是否配置正确。

## 9. 安全规范

1. 调试时速度建议不超过 `0.4`。
2. 每段动作结束后发送 `stop`。
3. 网页关闭或游戏结束时发送 `stop` 或 `lie`。
4. 保持机器狗周围空旷。
5. 不要让多个程序同时控制机器狗。
6. 保留原厂遥控器作为现场安全兜底。
7. 出现异常动作时立即发送 `emergency` 或使用原厂遥控器停止。

## 10. 交付给客户时建议说明

建议将客户二次开发入口说明为：

1. 做本地算法或传感器项目：使用 C++ SDK。
2. 做网页互动、教学、小游戏：使用 WebSocket 网页控制。
3. 不建议客户直接操作机器狗 UDP 原始协议，除非需要扩展底层能力。
4. 客户新增网页项目时放在 `web/` 下，通过 `ws://<控制板 Wi-Fi IP>:9001/ws` 控制机器狗。
5. 客户新增 C++ 项目时参考 `examples/`，链接 `lib/libquadruped_sdk.a` 并包含 `include/quadruped_sdk/`。
6. 如果需要批量部署到其他控制板，可参考 `docs/机械狗Docker部署指南.md` 构建和运行 Docker 镜像。
