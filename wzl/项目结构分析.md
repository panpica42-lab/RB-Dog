# quadruped_dev_sdk 项目结构分析

本文档是放在 `wzl/` 里的临时沟通记录，方便后续我和你协作。交付前可以手动删除整个 `wzl/` 文件夹，不影响原始交付包。

## 这是什么包

这个大目录是一个机械狗二次开发交付包，核心由几块组成：

- C++ SDK：封装和机械狗 UDP 协议通信、JSON 协议解析、状态查询、动作控制、IMU 数据读取。
- Python WebSocket/UDP 网关：把网页/App 的 WebSocket 控制消息转成机械狗 UDP 协议，同时提供网页控制台和可选 RealSense RGB/点云转发。
- 网页控制台：放在 `web/`，由 Python 网关直接作为静态页面提供。
- 手机端上位机：`hbuilder_quadruped_controller/`，是 HBuilderX/uni-app 项目，用于打包 Android APK，连接同一个 WebSocket 网关做简单遥控。
- 部署材料：Docker、systemd、安装脚本、中文部署/二开文档。
- 机械结构文件：`CS-DOG1.STEP`，看起来是机械狗或相关结构的 STEP 三维模型文件。

## 根目录文件

- `README.md`：项目主说明，包含网络连接、编译、示例运行、WebSocket 网关、API 和安全注意事项。当前在 Windows PowerShell 输出中中文显示成乱码，VS Code 固定 `files.encoding: utf8` 后有助于避免编辑器误判编码。
- `Makefile`：主构建入口。负责编译静态库、示例程序，并提供 `make run_ws_gateway`、`make run_ws_mock`、Docker 构建/运行等目标。
- `Dockerfile`、`docker-compose.yml`、`.dockerignore`：容器化部署相关。
- `CS-DOG1.STEP`：三维结构/机械模型文件，体积较大。

## C++ SDK 部分

- `include/quadruped_sdk/`
  - `udp_client.h`：UDP 客户端封装，负责初始化 socket、发送/接收消息、超时和连接测试。
  - `json_parser.h`：SDK 的主要控制接口，定义消息类型、查询码、命令码、运动模式、状态结构，并提供复位、站立、趴下、急停、切换模式、速度控制、IMU 流等方法。
  - `imu_data.h`：IMU 数据结构，包含时间戳、线加速度、角速度、四元数、协方差和欧拉角计算接口。
- `src/`
  - `udp_client.cpp`、`json_parser.cpp`、`imu_data.cpp`：上述头文件对应实现。
- `examples/`
  - `example.cpp`：自动动作/基本控制示例。
  - `keyboard_test.cpp`：键盘控制示例。
  - `imu_client_example.cpp`：IMU 数据读取示例。
- `lib/`
  - `libquadruped_sdk.a` 和若干 `.o`：已经编译好的库产物。
- `bin/`
  - `example`、`keyboard_test`、`imu_client_example`：已经编译好的示例程序。

这一部分按 Linux 环境设计，头文件里直接包含了 `<sys/socket.h>`、`<netinet/in.h>`、`<arpa/inet.h>`、`<unistd.h>` 等 Linux/POSIX 头文件，所以在 Windows 本机 VS Code 里会报找不到系统头文件。已通过 `.vscode/settings.json` 关闭 C/C++ 扩展错误波浪线，同时把本项目 `include/`、`src/`、`examples/` 加入索引路径。

## 网关和网页控制台

- `tools/ws_gateway.py`：核心网关程序。
  - 监听 HTTP/WebSocket，默认端口 `9001`。
  - 静态托管 `web/` 下的网页控制台。
  - 将网页/App 发来的 `reset`、`stand`、`lie`、`move`、`emergency`、`model`、`obstacle`、`action`、`video`、`pointcloud` 等命令转成机械狗 UDP 协议。
  - 支持 `--mock` 模拟模式，无机械狗时返回模拟状态和 IMU。
  - 可选接入 RealSense，向客户端广播 RGB 图像和点云数据。
- `tools/capture_dog_udp.py`：从名字看是 UDP 抓包/记录工具，用于捕获机械狗通信数据。
- `web/index.html`：浏览器端控制台入口，由 `ws_gateway.py` 直接提供访问。
- `captures/*.jsonl`：已有的 UDP/协议捕获记录，适合后续分析协议或回放调试。

## 手机端上位机

- `hbuilder_quadruped_controller/` 是一个独立的 HBuilderX/uni-app 项目。
- `App.vue`、`main.js`、`pages.json`、`manifest.json` 是 uni-app 标准入口和配置。
- `pages/index/index.vue` 是主要界面和逻辑，包含：
  - 首屏连接页，默认控制器地址为 `10.0.50.236:9001`。
  - WebSocket 连接 `ws://<控制器地址>/ws`。
  - 收到网关 `hello` 后进入控制页。
  - 基本控制：站立、趴下、回零、急停、停止、模式切换、动作库、停障、视频开关。
  - 双摇杆控制：前后、左右、转向三轴速度。
  - 状态显示：电量、姿态/运动模式、模型、故障、日志。
  - 视觉页：RGB 图像和点云查看，点云支持旋转、缩放、复位。
- 这个 App 不直接和机械狗 UDP 通信，而是通过 `tools/ws_gateway.py` 暴露的 WebSocket 网关间接控制。

## 当前我已改的配置

已新增 `.vscode/settings.json`，主要作用：

- `"files.encoding": "utf8"`：让 VS Code 按 UTF-8 打开/保存文件，减少中文被误判编码的概率。
- `"C_Cpp.errorSquiggles": "disabled"`：关闭 C/C++ 扩展错误波浪线，避免 Windows 下 Linux 系统头文件找不到时满屏红线。
- `"C_Cpp.default.compilerPath": "/usr/bin/g++"`：告诉 C/C++ 扩展按 Linux GCC 理解项目。
- `"C_Cpp.default.intelliSenseMode": "linux-gcc-x64"`：IntelliSense 使用 Linux/GCC x64 模式。
- `"C_Cpp.default.cppStandard": "c++11"`：按 C++11 标准解析。
- `includePath`/`browse.path`：至少能找到本项目自己的 `include/`、`src/`、`examples/`。

## 后续可关注点

- 如果要真正编译运行 C++ SDK，建议在 Linux/控制板/Docker/WSL 中执行，而不是直接在 Windows 原生环境编译。
- 如果 VS Code 仍然显示中文乱码，需要确认文件本身是否已经被错误编码保存过；当前设置只能防止编辑器继续误判，不能自动修复已损坏文本。
- `hbuilder_quadruped_controller` 默认 IP 写死为 `10.0.50.236:9001`，客户现场网络变化时需要在 App 首屏或设置里改地址。
